Custom Macro Keyboard

Project Overview

A fully programmable mechanical keyboard featuring OLED displays for dynamic key labeling, rotary encoders for precise control, and hot-swappable mechanical switches. Designed specifically for content creators and professionals who need customizable shortcuts and visual feedback for their workflows.

Hardware Components

  • Raspberry Pi Pico (RP2040)
  • Cherry MX Brown Switches (x12)
  • OLED Displays (0.96" x 4)
  • Rotary Encoders (x2)
  • Custom PCB with Kailh sockets
  • RGB LED Strip (WS2812B)
  • 3D Printed Case (PLA+)
  • Aluminum Top Plate
  • USB-C Connector
  • Magnetic Feet for Stability
  • Custom Keycaps (Resin Printed)
  • Diodes and Resistors

Design & Development

  1. Requirements Analysis: Surveyed content creators for desired features
  2. Schematic Design: Created electrical design with switch matrix and OLED controllers
  3. PCB Layout: Designed compact PCB with optimal signal routing
  4. 3D Modeling: Created ergonomic case design in Fusion 360
  5. Firmware Development: Programmed in CircuitPython with custom HID library
  6. Software Suite: Built configuration tool for easy customization
  7. Prototyping: Multiple iterations of case and PCB refinements
  8. User Testing: Gathered feedback from beta testers

Key Features

Dynamic Key Labels

OLED displays show current function of each key, changing based on active application or mode.

Hot-Swappable Switches

Easy switch replacement without soldering for personalized feel and sound.

Multi-Layer Functionality

Multiple layers accessible via modifier keys for expanded functionality in compact form.

RGB Lighting

Customizable per-key RGB lighting with multiple animation modes and profiles.

Technical Challenges

Challenge: OLED Refresh Rate

Multiple OLED displays caused performance bottlenecks.

Solution: Implemented asynchronous display updates and frame buffering.

Challenge: Power Management

RGB LEDs and OLEDs drew significant power over USB.

Solution: Added intelligent brightness control and sleep modes.

Lessons Learned

  • Ergonomics are crucial for devices used for extended periods
  • User configuration tools are as important as the hardware itself
  • Manufacturing tolerances significantly impact assembly experience
  • Open-source documentation accelerates community adoption
  • Iterative prototyping saves time in the long run
